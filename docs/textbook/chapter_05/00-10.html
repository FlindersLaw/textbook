Complex Data Concepts

We already know that we can store stuff in variables. For example, we can store:

StringsMJFname = 'Mark Ferraretto'
IntegersMJFnumber_chn = 4
FloatsMJFhh_income = 45000.0
BooleansMJFis_citizen = True

What each example has in common is that each stores one piece of information ie:

  • one string;
  • one integer;
  • one floating point number; and
  • one boolean.

Sometimes, though, we might have data that looks like this:

A list of names
Name
Ida Noe
Hugo First
Moda Lawn
Kate Jupp
Ayfor Gott

or this:

How do we store this data? For example, let's look at our list of names above.

One way would be to store each name in a variable. We know we can store strings in variables, so maybe we could set up variables as follows:

MJFname1 = 'Ida Noe'
MJFname2 = 'Hugo First'
MJFname3 = 'Moda Lawn'
MJFname4 = 'Kate Jupp'
MJFname5 = 'Ayfor Gott'
MJFname6 = 'Justin Thyme'
MJFname7 = 'Ray Member'
MJFname8 = 'Studie Harde'
MJFname9 = 'Slacke Hearse'

Technically this works, but it raises some issues:

First, it's not going to scale very well. Imagine needing to manage a list of 100, or 1,000 names. We'd need 100 or 1,000 variables. This is a lot of work and, specifically, a lot of repetitive work. It's inefficient, time-consuming and error-prone.

Wouldn't it be nice if we could store all these names in a single variable regardless of how many we had...

Also, how would we manage displaying all the names? For example, how would we display the list above? We'd have to list out each MJFnameN variable each time. What happens when we add or delete someone? We need to create new variables and copy them throughout our code. When we delete names we'd have gaps in our sequencing. How would we know the gap is there on purpose and not an error?

This is unworkable, much like carrying 10 buckets around each with one thing inside, when what you really want is to carry one bucket around with 10 things inside.

If only we had a way of displaying all the names using a single variable and just added and removed names from it as we needed to.

If only we could be as happy as the Happy Bucket Lady...

Introducing Complex Data Structures​

Complex data types are the answer to our problem. Complex data types, such as lists and dictionaries give us Happy Bucket Lady vibes. This is because complex data types allow us to store more than one value in a variable. Even more so, complex data types also allow us to structure how we will store the data. For example, a list complex data type allows us to store data in sequential order, much like that 'List of names' table above. Because complex data type allow us to do this, they are often also referred to as complex data structures. We will use both terms interchangeably in this textbook.
