Variables

We covered variables briefly in Chapter Two but let's go over them again in a bit more detail.

What is a Variable?​

A variable is a place where you store stuff. It's called a 'variable' because you can store different stuff in ther each time. We've already seen that we can run the same program many times, each time with different values.

Variables are like buckets​

We've said this before and we'll say it again.

A variable is kind of like a bucket. You can put stuff in a bucket and you can put stuff in a variable.

VariableBucket
Used to store stuffUsed to store stuff
Must have a nameDoesn't usually have a name
Can store (assign) stuff in itCan store stuff in it

For example, we've been using MJFfirstname to store the first name in our code in Chapter Two. Each time we run the program we're asked for our first name. We can enter what we like and each time we run the program we can enter something different if we want. Each time we enter a first name into the app, what we enter is stored in MJFfirstname.

In Python-speak, what we store in a variable is called its value. The term 'value' will be used a lot in this textbook.

Let's have a look at the second question block in this code. It looks like this:

---
# My first Docassemble program
mandatory: True
question: Hello World!
...
---
question: Name
fields:
  - Salutation: MJFsalutation
    choices:
      - Dr
      - Mr
      - Mrs
      - Ms
  - First Name: MJFfirstname
  - Last Name: MJFlastname
---

You'll recall that this code displays this form:

The second-last line of the code...

  - First Name: MJFfirstname

...tells Docassemble to display the 'First Name' field in the form. The user will enter something into the 'First Name' text field. When the user presses continue Docassemble will store, or assign the value the user enters to the variable MJFfirstname. Once that assignment is made, then we can access what was assigned by calling MJFfirstname, as we do in line 24 of the same code:

---
# My first Docassemble program
mandatory: True
question: Hello World!
subquestion: |
...
  Column1|Column2
  -------|-------
  Salutation|${ MJFsalutation }
  First Name|${ MJFfirstname }
  Last Name|${ MJFlastname }
tip

We usually use question blocks to display forms and collect data and assign that data into variables.

We will then access the data in variables to display information on a screen (by using a subquestion). We will also usually use Mako in the subquestion tag to access those variables.

Terminology Recap​

We've introduced lots of new concepts, so it's worth revisiting these.

Variable​

A variable is what we use to store data. Much like how we put stuff in a bucket, we can put stuff in a variable.

Value​

Value is another way of saying 'stuff', or data. Data stored in a variable is referred to as that variable's value.

Accessing a variable's value​

To access a variable's value we simply specify the variable's name. For example, we're displaying the value stored in each of MJFsalutation, MJFfirstname and MJFlastname in this code from the subquestion block above:

  Column1|Column2
  -------|-------
  Salutation|${ MJFsalutation }
  First Name|${ MJFfirstname }
  Last Name|${ MJFlastname }

Assignment​

The process of putting a value into a variable is called assignment. We assign a value to a variable. We can assign a value to a variable in two ways.

Asking the user a question​

When we set up fields in a question block, the user is asked to input what values they think should go into one or more variables. For example, we use this question block to ask the user how they should be addressed, their first and their last name.

---
question: Name
fields:
  - Salutation: MJFsalutation
    choices:
      - Dr
      - Mr
      - Mrs
      - Ms
  - First Name: MJFfirstname
  - Last Name: MJFlastname
---

Docassemble displays this form:

When the user presses Continue Docassemble assigns the values entered by the user into MJFsalutation, MJFfirstname and MJFlastname respectively.

Assignment in code blocks​

We can also assign values to variables in code blocks.

For now, it's just worth being aware that you can also assign variables in python code. We use the equals sign (=) to show that a value is being assigned to a variable. For example:

code: |
  MJFname = 'Mark Ferraretto'
note

Because = is used as an assignment variable we can't use it to test for equality. That is, the above code block sets the value of MJFname to be Mark Ferraretto. It is not a test to see if the value in MJFname is equal to Mark Ferraretto.
To test that something equals something else we need to use the 'double-equals' operator (==).

Compare these examples:

# Assigns 'Mark Ferraretto' to MJFname
code: |
  MJFname = 'Mark Ferraretto'
# Tests to see if the value of MJFname is equal to Mark Ferraretto
code: |
  MJFname == 'Mark Ferraretto'

Be mindful of this as confusing = and == is a common mistake!

We will cover testing further down in this chapter.
