Docassemble Syntax

tip

You may want to book this part of the textbook. As you learn to write code you can come back here to check that your syntax is correct

Docassemble's syntax is heavily dependent on where code appears in a file.

You should ensure that:

  • All tags and block separators must start at the left margin.
  • All content following a : | must be indented by two spaces. Docassemble will not accept, one or three or five. It must be two spaces.
  • All tags are case sensitive. Question: is not the same as question:. If you input Question: Docassemble will generate a syntax error. Case is important in Docassemble.
  • There must be a space between a : and a |. That is, subquestion: | is fine. But subquestion:| (no space), subquestion: | (two spaces) and Subquestion: | (two spaces but a capital 'S') will all generate syntax errors.

Docasemble is very fussy with syntax. If you get errors when writing your code, you should check syntax first. Make sure your spaces, cases, braces, brackets, colons, vertical bars, dashes are all correct.

tip

Pressing the Tab key in the text editor automatically indents your text for you by two spaces.

General Principles​

Below are some extracts of Docassemble code. Don't worry if you don't understand what the code is doing. We're just using this code to demonstrate how Docassemble uses spacing and indents to help it work out what the code you write means.

Just focus on the spaces...

  • Each block starts and finishes with --- which must be aligned against the left margin.
  • The primary 'tags' or 'specifiers' that set out what is in a block must also be aligned against the left margin
  • Each specifier must finish with a colon (:) and then be followed with text (depends on the specifier)
  • Text that follows a specifier must be separated by a space after the colon
  • Text in a subquestion block must be indented by two spaces from the left margin
  • Commands that display elements (such as buttons in this example) must be indented by two spaces.
  • Syntax for these commands is:
    • Indent by two spaces
    • a dash (-)
    • one more space
    • A label (the text that will be displayed on a button in this case)
    • a colon (:) and then a space
    • The command or variable. In this case we're describing buttons so the text is the action that Docassemble should take when the button is pressed

Code blocks​

We haven't covered what code blocks are yet but don't worry. Just read through the spacing and keep it in your mind for when we do cover code blocks.

  • The content of a code block is python code.
  • All code starts by being indented two spaces from the margin
  • Blank lines can be inserted in the code to aid in readability
  • This code displays some 'if' statements. These are discussed further in Chapter Four.
  • The true branch and false branch of an if statement must be indented by a further two spaces
  • An 'if' or 'else' in the code block must be terminated with a colon
